1. Ringkasan Proyek

Media Kajo adalah platform portal berita digital berbasis web yang memungkinkan tim redaksi menerbitkan artikel dari berbagai kategori (Teknologi, Bisnis, Olahraga, Gaya Hidup) kepada pembaca umum. Sistem dibangun dalam dua bagian utama: Public News Portal yang dapat diakses publik, dan Admin CMS Dashboard untuk manajemen konten internal.

Proyek ini dirancang sebagai produk startup yang siap production — clean architecture, skalabel, dan portfolio-ready — menggunakan stack modern berbasis React + Node.js + PostgreSQL.

🎯 Target Produk

Portal berita digital dengan CMS internal untuk publikasi konten multi-kategori

👤 Target Pengguna

Pembaca umum (portal) + Tim redaksi/admin (CMS dashboard)

⚡ Stack Utama

React · Tailwind · Node/Express · PostgreSQL · Prisma

2. Tujuan & Sasaran

Business Goals

  • Menyediakan platform publikasi berita yang dapat dioperasikan tim kecil (1–5 editor)
  • Membangun kehadiran digital Media Kajo yang profesional dan terpercaya
  • Menyiapkan fondasi teknis yang siap untuk monetisasi (iklan, langganan) di masa depan

Product Goals

  • Pengalaman baca yang bersih, cepat, dan nyaman di desktop maupun mobile
  • Workflow publikasi yang intuitif bagi editor non-teknis
  • Performa halaman baik (target LCP < 2.5 detik)
  • Arsitektur yang mudah dikembangkan tanpa refactor besar

Out of Scope (v1.0)

  • Sistem komentar publik
  • Notifikasi push / newsletter
  • Sistem langganan berbayar (paywall)
  • Multi-bahasa (internasionalisasi)
  • Aplikasi mobile native
Prinsip desain produk: Avoid overengineering. Bangun yang dibutuhkan sekarang dengan arsitektur yang memudahkan ekstensi nanti — bukan sebaliknya.

4. Public News Portal

Bagian yang dapat diakses oleh seluruh pengguna tanpa login. Fokus pada pengalaman membaca yang bersih dan cepat.

4.1 Halaman & Fitur

🏠

Homepage

Hero section dengan artikel featured, grid artikel terbaru, navigasi kategori horizontal, dan section "Trending" berdasarkan artikel terpopuler.

📋

News List

Daftar artikel dengan filter kategori, sorting (terbaru/terpopuler), dan pagination. URL berbasis query string agar shareable.

📄

Article Detail

Tampilan artikel penuh: judul, thumbnail, konten rich text, nama penulis, tanggal publish, kategori badge, dan artikel terkait di bawah.

🔍

Search

Full-text search berdasarkan judul dan isi artikel. Hasil ditampilkan real-time dengan highlight keyword. Dapat diakses via search bar di navbar.

4.2 Komponen UI Global

Navbar
  • Logo Media Kajo (kiri)
  • Category links — dinamis dari database, maksimal 6 kategori di navbar
  • Search icon/bar — expand on click di desktop, full-width di mobile
  • Responsive: hamburger menu di mobile (< 768px)
News Card
  • Thumbnail — rasio 16:9, lazy loaded, fallback placeholder
  • Category badge — warna per kategori
  • Judul — max 2 baris, font serif
  • Excerpt — max 3 baris terpotong
  • Meta — nama penulis · tanggal publish
Footer

Branding, link kategori, copyright. Simple dan minimal.

4.3 Responsiveness

5. Admin CMS Dashboard

Interface internal untuk tim redaksi. Diakses via rute /admin, terproteksi dengan JWT authentication.

5.1 Halaman & Fitur

🔐

Login Page

Form email + password. Validasi client-side. JWT token disimpan di httpOnly cookie. Redirect ke dashboard setelah berhasil login.

📊

Dashboard Overview

Metric cards: total artikel, total kategori, artikel bulan ini, artikel draft. Tabel 5 artikel terbaru dengan quick actions.

✍️

Manajemen Artikel

Tabel artikel dengan kolom: thumbnail mini, judul, kategori, penulis, status, tanggal. Filter & sort. Bulk delete. Aksi: Edit / Delete / View.

📝

Form Artikel

Form create/edit: judul, slug (auto-generated), kategori dropdown, thumbnail upload, rich text editor (konten), status (Draft/Published), tanggal publish.

🏷️

Manajemen Kategori

CRUD kategori: nama, slug, warna badge, deskripsi singkat. Validasi nama duplikat. Preview badge live.

👤

Profile Settings

Edit nama, email, password admin. Simple, satu halaman.

5.2 Rich Text Editor

Menggunakan Tiptap (atau Quill.js sebagai alternatif) untuk WYSIWYG editing. Fitur yang didukung:

  • Heading (H1–H3), Bold, Italic, Underline
  • Ordered/Unordered list
  • Blockquote, Code block
  • Insert image (upload atau URL)
  • Link dengan target blank

5.3 Upload Gambar

Upload thumbnail menggunakan Multer di backend. Gambar disimpan di folder /uploads lokal (v1.0) dengan rencana migrasi ke cloud storage (S3/Cloudinary) di v1.1. Validasi: max 5MB, format JPG/PNG/WebP.

Catatan: Untuk production, gambar wajib dipindahkan ke CDN/cloud storage agar tidak membebani server aplikasi. Flag ini sebagai technical debt v1.0 yang harus diselesaikan sebelum traffic signifikan.

10.3 Pola Arsitektur

  • Monolith Modular — satu codebase backend tapi terstruktur per domain (article, category, auth). Mudah dipecah ke microservice jika diperlukan nanti.
  • REST API — JSON-based, versioned dengan prefix /api/v1/
  • Separation of concerns: Route → Controller → Service → Prisma. Controller hanya handle request/response, logic di service layer.

13. Security

Authentication & Authorization

  • JWT disimpan di httpOnly cookie (bukan localStorage) — aman dari XSS
  • Token expiry: 7 hari, refresh via re-login
  • Semua route admin dilindungi middleware authMiddleware yang validasi token sebelum controller berjalan
  • Password di-hash dengan bcrypt, saltRounds minimum 12

Input Validation & Sanitization

  • Semua request body divalidasi dengan Zod schema di level middleware
  • Konten HTML dari rich text editor di-sanitize dengan DOMPurify sebelum disimpan
  • File upload: validasi MIME type dan ekstensi, size limit 5MB, rename file random

API Security

  • CORS dikonfigurasi hanya mengizinkan origin frontend yang terdaftar
  • Rate limiting pada endpoint auth: max 10 request/menit per IP
  • Helmet.js untuk HTTP security headers
  • SQL injection: tidak mungkin karena Prisma menggunakan parameterized queries
Sebelum production: Audit environment variables (jangan commit .env), pastikan HTTPS aktif, review semua exposed route, dan lakukan dependency audit (npm audit).

16. Risks & Constraints

RiskLevelMitigasi
File upload lokal tidak scalable Medium Tandai sebagai tech debt, migrasi ke S3/Cloudinary di v1.1 sebelum traffic tinggi
Search LIKE query lambat saat artikel banyak Medium Gunakan PostgreSQL full-text search dengan GIN index. Elasticsearch bisa ditambahkan nanti.
JWT token tidak bisa di-revoke (stateless) Low Expiry pendek (7 hari). Jika butuh revoke instan, tambahkan token blacklist di Redis (v1.1)
Rich text output rentan XSS High Wajib sanitize HTML dengan DOMPurify sebelum render dan sebelum simpan ke DB
Single admin user (no role system) Low Desain schema siapkan field role di tabel User untuk ekstensi nanti
Scope creep selama development Medium Strict dengan Must Have scope di v1. Feature request baru masuk ke backlog v1.1

Constraints

  • Tidak ada budget untuk external services berbayar di v1.0 (file storage lokal, PostgreSQL gratis tier)
  • Tim kecil: 1–2 developer, sehingga scope Must Have harus realistis
  • Target launch dalam 10 minggu — tidak ada waktu untuk over-engineering
